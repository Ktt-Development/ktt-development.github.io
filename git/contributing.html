Contributing

Basic Guidelines

Use English (US).

Files, methods, variables, and comments* must be written in English.

*comments may be written in any language but must have an English translation

Use an IDE.

It is recommended to use IntelliJ or PyCharm.

Make sure compile is stable.

Do not submit a pull request without testing if the code compiles.

Do not make large changes.

Large changes are unlikely to be reviewed in a timely matter, pull requests should be focused on one issue or feature.

Associate pushes and pull requests.

Associate with issue numbers.

Style Guidelines Java

  • Use features only in JDK 11.
  • Use 4 space indentation.
  • Do not wrap lines (let the IDE wrap lines visually)
Imports
  • Use static imports for enums that have more than one value.
  • Use static imports for enums within clases.
Classes
  • Use

    PascalCase

    for class and package names.
  • Do not start with _ or use $ in class names
  • Keep scope and access modifiers in the same order.
  • Always use

    final

    keyword when possible.
Variables
  • Use

    camelCase

    when possible.
  • Do not start with _ for variable names.
  • Keep scope and access modifiers in the same order.
  • Always use

    final

    keyword when possible.
  • Array assignment should be multiline except when using primitive data types.
  • Use primitive data types except where required (like lists).

    Integer int Double double Boolean boolean Char char

  • Assign overused variables in a higher scope.
Methods
  • Use

    camelCase

    for method names.
  • Do not start with _ for method names.
  • Keep scope and access modifiers in the same order.
  • Methods inside constructors should not use

    this

    unless necessary
  • Parameters must always be

    final

    .
Syntax
  • Organize sections of code using regions
  • Use double brace instantiation.
  • No surrounding spaces.
  • Keep variable assignment in columns (except in one).
  • Use simplified expressions.
  • Keep lengthy statements on multiple lines.
  • Use conditional expressions where possible.
  • Use switch cases instead of if statements where possible.
  • Use for each loops unless index is necessary.
Files
  • Use only lower case file extensions.
