General Guidelines

Use English (US).

Files, methods, variables, and comments* must be written in English.

*comments may be written in any language but must have an English translation.

Use an IDE.
  • Java
  • Python

It is recommended to use IntelliJ.

It is recommended to use PyCharm.

Make sure that the code compiles.

Do not submit a pull request without testing if the code compiles first.

Do not make large changes.

Large changes are unlikely to be reviewed. If you are introducing a major change please have multiple pull requests for individual changes to your own branch, then submit a pull request for the final; This will make tracking changes easier.

Include issue numbers with commits and pull requests.

This will make tracking issues easier.

Code Style Guidelines

  • Use 4 space indentation.
  • Java
  • Python

Use features only in JDK 11.

Use features only in Python 3.8

Imports
  • Java
  • Use static imports for enums that have more than one value.
  • Use static import for enums within classes.
Classes
  • Java
  • Use

    PascalCase

    for class and package names.
  • Do not start with _ for variable names.
  • Keep scope and access modifiers in the same order.
  • Always use

    final

    keyword when possible.
Variables
  • Java
  • Use

    camelCase

    when possible.
  • Do not start with _ for variable names.
  • Keep scope and access modifiers in the same order.
  • Always use

    final

    keyword when possible.
  • Array assignment should be multiline except when using primitive data types.
  • Use primitive data types except where required (like lists).
  • Assign overused variables in a higher scope where possible.
Methods
  • Java
  • Use

    camelCase

    for method names.
  • Do not start with _ for method names.
  • Keep scope and access modifiers in the same order.
  • Methods inside constructor should not use this unless necessary.
  • Parameters must always be

    final

    .
  • Do not create unnecessary methods. Use a

    Runnable

    or

    Consumer

    when possible.
  • Use lambdas where possible.
Syntax
  • Java
  • Use double brace instantiation.
  • No surrounding spaces.
  • Remove brackets for single line conditionals.
  • Use shorten operators and keep spaces between operations.
  • Use ternary expressions where possible.
  • Use switch cases instead of if statments where possible
  • Use for each loops unless index is necessary.
